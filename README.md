# law-map（法令星図 / Law Constellation）

兵庫県建築基準条例から見上げる、純粋な法令のマップ。建築・災害ドメインの日本法令を「参照関係」で結んだインタラクティブな星図。日本語圏に公開先行例のない法令ネットワーク可視化の空白を、小さく確実な範囲で埋める試作（portfolio / ゼミ手土産枠）。

**ライブデモ**：https://shg195.github.io/law-constellation/web/ （GitHub Pages 有効化後に閲覧可）

ノードは法的文書（法令・条・規則・判例）のみ。desc/src/rel は中立の書誌事実のみを載せる（分析・考察・物語は載せない）。

## 何であるか

- **A層（機械抽出）**：e-Gov 法令API v2 で法令XMLを取得し、本則テキスト中の「〜法第◯条」等の法令間参照を抽出したネットワーク。ノード=法令、エッジ=参照（太さ=参照回数）。シード13法令から2ホップ探索し上限150法令。
- **B層（手作業・一次資料移植）**：兵庫県建築基準条例（31条を条単位で全収載）＋神戸市・東京都・奈良県ため池の各条例＋関連判例＋建築基準法の委任条項を、一次資料から逐語で起こした層（全70ノード）。A層とは `bridge`（同一法令）で接続。
  - 兵庫県建築基準条例の条ノードには、条文本文（逐語・解説文は除く）を `text` フィールドとして収載する（出典＝『兵庫県建築基準条例及びその解説』令和7年4月1日版）。表を含み行の再構成に信頼性リスクがある条（第2条の2）及び原文が確認できない条（第21条）は未収載。
- **ビュー**：既定は `2D 断面`（法段階を縦軸にした重力井戸の断面・d3）。3Dは実験モード（`?mode=3d` をURLに付与した場合のみ・情報量は2Dと同一）。
- **多軸フィルタ（Legislative Explorer 輸入）**：法段階・年代（元号）・B層表示ON/OFF・エッジ種別（凡例クリックでトグル）。
- **スマホ対応**：ピンチズーム／パンのタッチ操作に対応。狭幅（〜430px）ではフィルタが折りたたみ式（既定閉）、詳細パネルは画面下部のシート表示になる。

## データの出自と限界（重要）

情報の正確性を最優先し、一次資料で裏取りできる事実だけを収録している。ただし試作段階ゆえ次の限界がある。**星図の見た目を「現在の強い依存関係」と読みすぎないこと。**

- **実質1ホップに近い**：2ホップは探索の上限であって生存の保証ではない。足切りはシード近接度（シード拡張からの被参照 distinct 法令数）で行うため、2ホップ目の法令が最終150に残るかはデータ次第（現状 level2 生存=5件）。
- **本則限定**：参照抽出は本則（MainProvision）のみを対象とする。附則（経過措置・改正付随）を含めると参照の過半が歴史的言及になり星図が歪むため除外した（附則混入約42%を実測・除去）。したがって附則に由来する依存は現れない。
- **照応・略称はスキップ**：「同法」「この法律」等の照応表現は指示先を解決していない（カウントのみ）。そのぶんエッジは実際の依存より少なめに出る。
- **A層は法律に偏る**：法令は他法令を「昭和◯年法律第◯号」型の公布番号で引くのに対し、施行令・省令は名称で引くのが通例で番号アンカーに乗りにくい。結果、府省令ノードはほぼ現れない（type分布：法律147／政令3）。
- **参照抽出の精度**：正規表現ベース。最長一致＋誤爆ガード2種を実装し、サンプル20件の手動照合で precision=100% を実測（検証手順は非公開）。ただし全数保証ではない。
- **エッジの既定表示**：全エッジ種別を既定ONで表示する（「参照（A層・機械抽出）」は全1783本中1693本＝約95%を占め本数が支配的だが、既定OFFにするとA層ノードの大半が孤立表示になるためON）。見やすさは非表示ではなく描画側で担う：a-ref（A層参照）は背景として薄く細く、B層・bridge・委任系エッジは前景として太く濃く描く（既定表示で孤立ノード0を実測）。フィルタの「エッジ種別」で個別にOFFにできる（全種別ONの状態は「すべて表示に戻す」で復元）。
- **B層の未確定判例2件**：東京高判H12.10.26・福岡地小倉支部R6.7.18 は判例集の正式な出典（巻号・頁・事件番号）が未確定のため、ノード詳細に「出典未確定・断定を避けています」の赤帯を必ず表示する（削除せず注記付きで残す方針）。
- **制定年**：A層は法令ID先頭3桁（元号1桁＋年2桁）から決定的に復号（era値と全150ノードで不一致0を実測）。推測ではない。

各ノードの出典（資料名＋頁）は詳細パネルに表示する。

## ローカル起動

依存を追加インストールせずブラウザで開ける（ライブラリは `web/lib/` にローカルバンドル済み・CDN依存なし）。リポジトリ直下で静的サーバを立てて `web/index.html` を開く。

```sh
# リポジトリ直下で
python -m http.server 8000
# → ブラウザで http://localhost:8000/web/index.html
```

`file://` 直開きは `fetch` がCORSで失敗するため不可。必ずHTTPで配信する。

### データパイプラインの再生成（任意）

```sh
python -m venv .venv
. .venv/Scripts/activate            # Windows(Git Bash)
pip install requests
PYTHONIOENCODING=utf-8 python pipeline/build_graph.py    # data/output/graph.json を再生成
PYTHONIOENCODING=utf-8 python pipeline/build_b_layer.py  # web/b_layer.json を再生成
```

e-Gov API はキー不要。初回は法令XMLを `data/cache/`（gitignore）にキャッシュし、再実行時はAPIを呼ばない。

## 構成

```
pipeline/   Python パイプライン（egov.py=APIクライアント / extract.py=参照抽出 / build_graph.py=グラフ生成 / build_b_layer.py=B層生成）
web/        静的Web（index.html / app.js / style.css / b_layer.json / lib=ローカルバンドル）
data/       output/graph.json（A層グラフ・リポジトリ同梱＝クローンだけで表示可）・output/references_raw.json（中間生成物・gitignore）・cache/（APIキャッシュ・gitignore）
```

## 出典・データソース

- e-Gov 法令API v2（`https://laws.e-gov.go.jp/api/2`・APIキー不要）— A層の法令名・種別・本文XML。
- B層移植元（一次資料）：兵庫県まちづくり部建築指導課『兵庫県建築基準条例及びその解説』（令和7年4月1日版・各条趣旨・頁対応）、洲永力・岡本和彦「がけ条例による建築規制の強度の実態に関する研究」（日本建築学会技術報告集 30(75), 2024）ほか各種判例集・官公庁資料を参照。
- 出典表記：デジタル庁 e-Gov 法令API（v2）を利用し、データを加工して作成。本サービスはデジタル庁が提供するものではありません。
